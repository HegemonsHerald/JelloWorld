


- you can press on the column beside the row numbers to show the current scope
- if you double-click on that space, you can set break-points and wath-points for the debugger

- run the debugger with the tiny 'bug' symbol
   - use the buttons at the top to step, step-into and jump, execute til breakpoint, etc
   - on the right you've got the break-point list, and variable and expression inspection tools
   - on the left you've got the bloody call stack baby!

   - in the Debug Perspective in the debug pane you can set conditionals and hit counts for
     break points to conditionally stop the program's execution!

   - you can also test an expression in the current state of the program's execution


- under window>perspective you can switch between the java and debug viewS
